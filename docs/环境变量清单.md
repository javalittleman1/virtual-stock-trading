# 环境变量清单

| 文档版本 | 修改日期   | 修改内容                 |
| -------- | ---------- | ------------------------ |
| V1.0     | 2026-04-09 | 初稿，定义全量环境变量   |

---

## 1. 概述

本项目基于 Next.js + Supabase + Vercel 架构，环境变量用于管理敏感信息（API 密钥、数据库连接串）及环境差异配置。所有变量均通过 `.env.local`（本地开发）和 Vercel 项目设置（生产环境）进行配置。

**命名规范**：
- 前端可暴露的变量以 `NEXT_PUBLIC_` 前缀开头。
- 服务端专用变量（如密钥）不加此前缀，仅在服务端代码中可用。

---

## 2. 环境变量列表

### 2.1 Supabase 相关

| 变量名                      | 必需性 | 说明                                                         | 示例值                                        | 获取方式                                   |
| --------------------------- | ------ | ------------------------------------------------------------ | --------------------------------------------- | ------------------------------------------ |
| `NEXT_PUBLIC_SUPABASE_URL`  | ✅ 必需 | Supabase 项目 URL，前端初始化客户端时使用                     | `https://abcdefghijklm.supabase.co`           | Supabase 项目设置 → API → Project URL      |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | ✅ 必需 | Supabase 匿名密钥，用于前端客户端鉴权（权限受 RLS 限制）        | `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`     | Supabase 项目设置 → API → anon/public key  |
| `SUPABASE_SERVICE_ROLE_KEY` | ✅ 必需 | Supabase 服务端密钥，**仅用于服务端**（绕过 RLS，拥有最高权限） | `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`     | Supabase 项目设置 → API → service_role key |
| `SUPABASE_DB_PASSWORD`      | ⚠️ 可选 | 数据库密码（若需直连 PostgreSQL，通常不必要）                 | `your_db_password`                            | Supabase 项目设置 → Database → Connection string |

> **注意**：`SUPABASE_SERVICE_ROLE_KEY` 切勿暴露给前端，仅用于 Next.js API 路由或服务端函数。

## 3. 本地开发配置

在项目根目录创建 `.env.local` 文件（已被 `.gitignore` 忽略），内容示例：

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

# iTick
ITICK_API_KEY=sk_live_xxxxxxxxxxxx

# 功能开关（可选）
NEXT_PUBLIC_ENABLE_ANALYSIS=false
NEXT_PUBLIC_INITIAL_BALANCE=1000000
NEXT_PUBLIC_TRADE_FEE_RATE=0.00025

# Cron 密钥（可选，若使用 Vercel Cron）
CRON_SECRET=your_random_secret_here
```

## 4. Vercel 生产环境配置

在 Vercel 项目 Dashboard → Settings → Environment Variables 中添加以下变量（与 `.env.local` 对应）：

| Key                              | Value                                       | Environment                |
| -------------------------------- | ------------------------------------------- | -------------------------- |
| `NEXT_PUBLIC_SUPABASE_URL`       | `https://your-project.supabase.co`          | Production, Preview, Dev   |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY`  | `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`   | Production, Preview, Dev   |
| `SUPABASE_SERVICE_ROLE_KEY`      | `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`   | Production, Preview, Dev   |
| `ITICK_API_KEY`                  | `sk_live_xxxxxxxxxxxx`                      | Production, Preview, Dev   |
| `CRON_SECRET`                    | `your_random_secret_here`                   | Production                 |
| `NEXT_PUBLIC_ENABLE_ANALYSIS`    | `false`（MVP） / `true`（正式）              | Production, Preview, Dev   |

> **注意**：`NEXT_PUBLIC_*` 变量在构建时被嵌入前端代码，更改后需重新部署生效。

---

## 5. 环境变量安全须知

1. **绝不要**将 `SUPABASE_SERVICE_ROLE_KEY` 或任何服务端密钥添加 `NEXT_PUBLIC_` 前缀。
2. 在本地开发时，`.env.local` 文件应被 `.gitignore` 忽略，防止意外提交。
3. 定期轮换 API 密钥（如 iTick），并在 Vercel 中更新。
4. 预览部署（Preview Deployments）会自动继承 Production 环境变量，除非单独覆盖。

---

## 6. 验证环境变量是否生效

可在 Next.js API 路由中打印检查（仅开发环境）：

```typescript
// app/api/health/route.ts
export async function GET() {
  return Response.json({
    supabase: !!process.env.NEXT_PUBLIC_SUPABASE_URL,
    itick: !!process.env.ITICK_API_KEY,
    cronSecret: !!process.env.CRON_SECRET,
  });
}
```

访问 `/api/health` 应返回各变量存在状态。
