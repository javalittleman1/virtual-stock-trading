当前个人投资者普遍缺乏系统化的交易训练环境，实盘操作面临较高的资金风险和心理压力。市面上现有模拟炒股软件存在数据延迟严重、用户体验陈旧、缺乏学习复盘功能等痛点。本项目旨在利用现代 Web 技术构建一款**轻量、实时、强教育属性**的模拟交易平台，帮助用户在零风险环境下学习股票交易、验证投资策略。 ### 1.2 产品定位 一款面向**个人投资者与金融爱好者**的 Web 端模拟炒股工具，提供近乎实盘体验的仿真交易环境，并内置交易分析与学习模块，实现“练习—复盘—成长”的闭环。同时支持电脑端（大屏高效操作）与手机端（移动便捷看盘）的无缝体验。 ### 1.3 核心价值 - **零门槛实盘模拟**：注册即获虚拟资金，完整复现真实市场交易规则。 - **即时反馈学习**：每笔交易自动记录，生成可视化收益曲线与行为分析报告。 - **极速开发部署**：基于 Supabase + Vercel 全栈架构，保障产品快速迭代与全球可用性。 - **跨端一致性**：一次开发，全端自适应，数据与操作状态在手机和电脑间实时同步。 --- ## 2. 目标用户 | 用户群体 | 特征描述 | 核心需求 | 典型使用场景 | | -------------- | ``` **stocks** （股票基础信息） ```sql CREATE TABLE stocks ( symbol TEXT PRIMARY KEY, name TEXT NOT NULL, market TEXT DEFAULT 'A', current_price DECIMAL(10,3), prev_close DECIMAL(10,3), open DECIMAL(10,3), high DECIMAL(10,3), low DECIMAL(10,3), volume BIGINT, updated_at TIMESTAMPTZ DEFAULT NOW() ); ``` **portfolios** （用户持仓） ```sql CREATE TABLE portfolios ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), user_id UUID REFERENCES profiles(id) ON DELETE CASCADE, stock_symbol TEXT REFERENCES stocks(symbol), quantity INT NOT NULL CHECK (quantity > 0), avg_cost DECIMAL(10,3) NOT NULL, created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW(), UNIQUE(user_id, stock_symbol) ); ``` **transactions** （交易记录） ```sql CREATE TABLE transactions ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), user_id UUID REFERENCES profiles(id) ON DELETE CASCADE, stock_symbol TEXT NOT NULL, type TEXT CHECK (type IN ('buy', 'sell')), quantity INT NOT NULL, price DECIMAL(10,3) NOT NULL, fee DECIMAL(10,3) DEFAULT 0, status TEXT DEFAULT 'filled', created_at TIMESTAMPTZ DEFAULT NOW() ); ``` **watchlist** （自选股） ```sql CREATE TABLE watchlist ( user_id UUID REFERENCES profiles(id) ON DELETE CASCADE, stock_symbol TEXT REFERENCES stocks(symbol) ON DELETE CASCADE, added_at TIMESTAMPTZ DEFAULT NOW(), PRIMARY KEY (user_id, stock_symbol) ); ``` ### 5.2 Row Level Security (RLS) 策略 ```sql -- 用户只能读写自己的数据 ALTER TABLE profiles ENABLE ROW LEVEL SECURITY; CREATE POLICY "Users can manage own profile" ON profiles USING (auth.uid() = id) WITH CHECK (auth.uid() = id); ALTER TABLE portfolios ENABLE ROW LEVEL SECURITY; CREATE POLICY "Users can manage own portfolios" ON portfolios USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id); ALTER TABLE transactions ENABLE ROW LEVEL SECURITY; CREATE POLICY "Users can read own transactions" ON transactions FOR SELECT USING (auth.uid() = user_id); ``` --- ## 6. 技术方案 ### 6.1 整体架构 ``` [用户浏览器] <--> [Vercel 边缘网络] <--> [Next.js 全栈应用] | ┌-----------------------+-----------------------┐ | | [Supabase] [第三方数据 API] (PostgreSQL + Auth + Storage) (iTick 等) ``` ### 6.2 技术栈选型 | 组件 | 技术选型 | 说明 | | ------------ | ----------------------------------------- | ---------------------------- | | 前端框架 | Next.js 15 (App Router) + TypeScript | 服务端渲染、API 路由一体化 | | UI 库 | shadcn/ui + Tailwind CSS | 高质量、可定制组件 | | 图表库 | Recharts / Lightweight Charts | K 线与分时图渲染 | | 后端服务 | Next.js API Routes + Vercel Edge Functions | 交易撮合、数据拉取 | | 数据库 | Supabase PostgreSQL | Realtime、Auth、Storage | | 认证 | Supabase Auth | 邮箱密码认证 | | 定时任务 | Vercel Cron Jobs | 定时拉取行情数据 | | 状态管理 | Zustand / Jotai | 轻量级全局状态 | --- ## 7.
