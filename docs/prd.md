# 模拟炒股软件 PRD

## 1. 产品概述

### 1.1 产品背景

当前个人投资者普遍缺乏系统化的交易训练环境，实盘操作面临较高的资金风险和心理压力。市面上现有模拟炒股软件存在数据延迟严重、用户体验陈旧、缺乏学习复盘功能等痛点。本项目旨在利用现代 Web 技术构建一款**轻量、实时、强教育属性**的模拟交易平台，帮助用户在零风险环境下学习股票交易、验证投资策略。

### 1.2 产品定位

一款面向**个人投资者与金融爱好者**的 Web 端模拟炒股工具，提供近乎实盘体验的仿真交易环境，并内置交易分析与学习模块，实现“练习—复盘—成长”的闭环。**同时支持电脑端（大屏高效操作）与手机端（移动便捷看盘）的无缝体验。**

### 1.3 核心价值

- **零门槛实盘模拟**：注册即获虚拟资金，完整复现真实市场交易规则。
- **即时反馈学习**：每笔交易自动记录，生成可视化收益曲线与行为分析报告。
- **极速开发部署**：基于 Supabase + Vercel 全栈架构，保障产品快速迭代与全球可用性。
- **跨端一致性**：一次开发，全端自适应，数据与操作状态在手机和电脑间实时同步。

## 5. 数据模型设计

### 5.1 核心表结构

> 以下表结构基于 PostgreSQL，并通过 Supabase 管理。**（同 V1.0，未做调整）**

**profiles** （用户扩展信息）
```sql
CREATE TABLE profiles (
  id UUID PRIMARY KEY REFERENCES auth.users ON DELETE CASCADE,
  email TEXT,
  virtual_balance DECIMAL(18,2) DEFAULT 1000000.00,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**stocks** （股票基础信息）
```sql
CREATE TABLE stocks (
  symbol TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  market TEXT DEFAULT 'A', -- A, HK, US
  current_price DECIMAL(10,3),
  prev_close DECIMAL(10,3),
  open DECIMAL(10,3),
  high DECIMAL(10,3),
  low DECIMAL(10,3),
  volume BIGINT,
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**portfolios** （用户持仓）
```sql
CREATE TABLE portfolios (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  stock_symbol TEXT REFERENCES stocks(symbol),
  quantity INT NOT NULL CHECK (quantity > 0),
  avg_cost DECIMAL(10,3) NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id, stock_symbol)
);
```

**transactions** （交易记录）
```sql
CREATE TABLE transactions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  stock_symbol TEXT NOT NULL,
  type TEXT CHECK (type IN ('buy', 'sell')),
  quantity INT NOT NULL,
  price DECIMAL(10,3) NOT NULL,
  fee DECIMAL(10,3) DEFAULT 0,
  status TEXT DEFAULT 'filled', -- filled, cancelled, partial
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**watchlist** （自选股）
```sql
CREATE TABLE watchlist (
  user_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  stock_symbol TEXT REFERENCES stocks(symbol) ON DELETE CASCADE,
  added_at TIMESTAMPTZ DEFAULT NOW(),
  PRIMARY KEY (user_id, stock_symbol)
);
```

### 5.2 Row Level Security (RLS) 策略示例

```sql
-- 用户只能读取自己的 profile
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users can view own profile" ON profiles
  FOR SELECT USING (auth.uid() = id);
CREATE POLICY "Users can update own profile" ON profiles
  FOR UPDATE USING (auth.uid() = id);
```

## 6. 技术方案

### 6.1 整体架构

```
[用户浏览器] <--> [Vercel 边缘网络] <--> [Next.js 全栈应用]
                                               |
                       ┌-----------------------+-----------------------┐
                       |                                               |
                 [Supabase]                                      [第三方数据 API]
            (PostgreSQL + Auth + Storage)                      (iTick / Yahoo Finance)
```

### 6.2 技术栈选型

| 组件         | 技术选型                                  | 说明                                                         |
| ------------ | ----------------------------------------- | ------------------------------------------------------------ |
| 前端框架     | Next.js 15 (App Router) + TypeScript      | 服务端渲染、API 路由一体化，完美契合 Vercel                  |
| UI 库        | shadcn/ui + Tailwind CSS                  | 高质量、可定制组件，开发效率高                               |
| 图表库       | Recharts / Lightweight Charts              | 轻量级，适合 K 线与分时图渲染                                 |
| 后端服务     | Next.js API Routes + Vercel Edge Functions | 处理交易撮合、数据拉取等逻辑                                  |
| 数据库       | Supabase PostgreSQL                        | 自带 Realtime 订阅、Auth、Storage                             |
| 认证         | Supabase Auth                              | 邮箱密码认证，后续可扩展社交登录                               |
| 定时任务     | Vercel Cron Jobs                           | 定时拉取行情数据，更新 stocks 表                              |
| 数据源       | iTick (A股) / Yahoo Finance (美股备用)     | 提供实时与历史行情数据                                        |
| 部署与托管   | Vercel                                     | 自动 CI/CD，全球 CDN                                          |
| 状态管理     | Zustand / Jotai                            | 轻量级状态管理，避免 prop drilling                            |

### 6.3 关键业务流程

#### 6.3.1 下单交易流程

1. 用户在前端填写股票代码、价格、数量，提交订单。
2. API 路由接收请求，通过 Supabase 服务端客户端校验：
   - 是否在交易时间（A 股 9:30-11:30, 13:00-15:00）
   - 是否涨跌停（市价单不可在涨跌停时下单）
   - 资金或持仓是否足够
3. 若为限价单，需判断当前行情是否满足价格条件（模拟撮合）。
4. 计算手续费，更新用户资金、持仓表，写入交易记录。
5. 返回成交结果，前端通过 Realtime 订阅更新界面。

#### 6.3.2 行情数据同步流程

1. Vercel Cron Job 每 30 秒触发一次 `/api/cron/update-prices` 端点。
2. 端点调用数据提供商 API 获取全量或重点股票实时价格。
3. 批量更新 Supabase `stocks` 表的 `current_price` 和 `updated_at`。
4. 由于 `stocks` 表开启了 Realtime，所有在线前端将自动收到更新并刷新 UI。

## 7. UI/UX 设计要点

### 7.1 通用设计原则

- **简洁专业**：主色调采用深色或蓝灰金融风格，突出数据可视化。
- **实时反馈**：价格变动时通过背景色闪烁（红涨绿跌）提供直观反馈。
- **响应式优先**：采用 Tailwind CSS 的响应式工具类，以移动端设计为起点，逐步增强至桌面端布局。

### 7.2 移动端专项设计 

| 设计要素         | 具体措施                                                     |
| ---------------- | ------------------------------------------------------------ |
| **导航**         | 底部标签栏（行情、自选、交易、资产、我的），符合拇指操作热区 |
| **行情卡片**     | 股票列表项采用紧凑卡片，仅展示代码、名称、最新价、涨跌幅，点击进入详情 |
| **图表交互**     | 支持单指拖动查看历史数据，双指缩放调整时间范围；横屏时自动切换全屏图表模式 |
| **交易操作**     | 买入/卖出使用底部动作面板，展示当前可用资金/持仓，输入框自动聚焦并调起数字键盘 |
| **加载与刷新**   | 下拉刷新获取最新行情，骨架屏占位提升感知性能                   |
| **手势操作**     | 自选股列表支持左滑删除；持仓列表支持长按查看更多操作           |
| **性能优化**     | 图片懒加载，非首屏图表延迟渲染，使用 `next/dynamic` 动态导入重型组件 |

### 7.3 桌面端专项设计 

| 设计要素         | 具体措施                                                     |
| ---------------- | ------------------------------------------------------------ |
| **多栏布局**     | 充分利用宽屏，左侧为导航菜单，中间为行情/图表主区域，右侧为持仓/交易面板 |
| **悬停提示**     | 图表数据点支持鼠标悬停显示精确数值；股票名称悬停显示详细信息卡片 |
| **快捷键**       | 支持键盘快捷键（如 `/` 聚焦搜索框，`Enter` 确认下单）          |
| **多窗口支持**   | 图表与交易面板可分离为独立窗口（浏览器多标签页），保持数据同步 |
| **精确操作**     | 所有输入框支持步进按钮，表格支持列排序与筛选                   |
