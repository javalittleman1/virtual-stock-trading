## 项目概述

这是一个基于Next.js构建的虚拟股票交易系统，采用实时价格更新机制来模拟真实的股票市场行情。系统通过Supabase提供实时数据订阅，结合定时任务实现价格数据的持续更新。

### 定时更新流程

系统采用分批轮询的方式实现价格更新，避免一次性请求过多股票导致API限流：

```mermaid

participant Supabase as Supabase数据库

Dashboard->>CronAPI : GET /api/cron/update-prices?offset=&limit=
CronAPI->>Supabase : 查询待更新股票列表
Supabase-->>CronAPI : 返回股票符号列表
CronAPI->>iTick : 批量获取股价数据

CronAPI->>Supabase : 更新股票价格字段

Dashboard->>Client : 触发实时订阅更新

```

## 性能优化策略

### 分批更新策略

系统采用分批更新机制，每次只更新3只股票，避免API限流：

- **批次大小限制**：每次最多3只股票
- **轮询更新**：通过offset参数实现循环更新
- **市场分组**：按沪市(SH)和深市(SZ)分别请求
- **延迟控制**：不同市场间添加2秒延迟

### 缓存与降级机制

```mermaid
flowchart TD
```

系统实现了多层次的错误处理和恢复机制：

```mermaid
stateDiagram-v2
[*] --> Normal : 正常运行
Normal --> APIError : API请求失败
Normal --> NetworkError : 网络连接异常
Normal --> TimeoutError : 请求超时
APIError --> Retry : 重试机制
NetworkError --> Retry
TimeoutError --> Retry
Retry --> Success : 重试成功
Retry --> Fallback : 回退到DB数据
Fallback --> Normal : 继续正常运行
Success --> Normal
APIError --> Alert : 发送告警
NetworkError --> Alert
TimeoutError --> Alert
Alert --> Normal
```

## 总结

该虚拟股票交易系统通过精心设计的实时价格更新机制，实现了接近真实市场的股票行情展示。系统的核心优势包括：

该机制为用户提供了一个流畅、准确的虚拟股票交易体验，同时为后续的功能扩展奠定了坚实的技术基础。
