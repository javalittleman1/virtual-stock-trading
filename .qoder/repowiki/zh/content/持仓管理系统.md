## 简介

虚拟股票交易平台的持仓管理系统是一个完整的资产管理解决方案，专注于提供实时的持仓跟踪、资产概览和交易执行功能。该系统基于Next.js构建，采用React状态管理，通过Supabase进行数据持久化，实现了从数据模型设计到前端展示的完整闭环。

系统的核心功能包括：
- 实时持仓数据管理与计算
- 资产概览与盈亏统计
- 股票价格实时更新机制
- 交易订单处理与资金管理
- 用户资产增减与历史记录追踪

### 资产概览计算

系统实现了完整的资产概览计算功能，包括总资产、可用资金、持仓市值和盈亏统计：

```mermaid
flowchart TD
Start([开始计算]) --> LoadData["加载持仓数据"]
LoadData --> CalcPosition["计算持仓市值<br/>position_value = Σ(quantity × current_price)"]
CalcPosition --> CalcCost["计算总成本<br/>total_cost = Σ(quantity × avg_cost)"]
CalcCost --> CalcAssets["计算总资产<br/>total_assets = available_balance + position_value"]
CalcAssets --> CalcProfit["计算盈亏<br/>profit_loss = position_value - total_cost"]
CalcProfit --> CalcPercent["计算盈亏百分比<br/>profit_loss_percent = (profit_loss / total_cost) × 100"]
CalcPercent --> CalcDaily["计算日收益<br/>daily_profit_loss = total_assets - initial_balance"]
CalcDaily --> CalcDailyPercent["计算日收益百分比<br/>daily_profit_loss_percent = (daily_profit_loss / initial_balance) × 100"]
CalcDailyPercent --> StoreResult["存储到状态管理"]
StoreResult --> End([结束])
```

### 实时价格更新机制

系统实现了定时价格更新机制，通过Cron任务定期从外部API获取最新股价：

```mermaid
flowchart TD
Start([定时任务触发]) --> CheckTime["检查交易时间"]
CheckTime --> TradingTime{"是否交易时间？"}
TradingTime --> |否| Skip["跳过更新"]
TradingTime --> |是| GetSymbols["获取股票代码列表"]
GetSymbols --> BatchProcess["分批处理股票"]
BatchProcess --> FetchBatch["批量获取股价"]
FetchBatch --> ValidateData["验证API响应"]
ValidateData --> UpsertData["批量更新数据库"]
UpsertData --> NextBatch{"还有批次？"}
NextBatch --> |是| FetchBatch
NextBatch --> |否| Complete["完成更新"]
Skip --> End([结束])
Complete --> End
```

## 性能考虑

系统在多个层面进行了性能优化：

### 数据缓存策略
- **实时订阅**：使用Supabase实时监听数据变更
- **批量更新**：股票价格更新采用分批处理减少API调用
- **状态缓存**：Zustand状态管理器避免不必要的重渲染

### 网络优化
- **请求去重**：避免重复的API请求
- **超时控制**：设置合理的请求超时时间
- **错误重试**：实现智能的错误处理和重试机制

### 前端性能
- **懒加载**：组件按需加载
- **虚拟滚动**：大量数据时使用虚拟化技术
- **防抖节流**：输入和搜索操作的性能优化

## 故障排除指南

### 常见问题及解决方案

**持仓数据不更新**
1. 检查Supabase连接状态
2. 验证实时订阅是否正常工作
3. 确认用户认证状态

**价格更新失败**
1. 检查Cron密钥配置
2. 验证API密钥有效性
3. 确认交易时间判断逻辑

**交易订单异常**
1. 检查交易规则验证
2. 确认资金余额充足
3. 验证股票代码有效性
